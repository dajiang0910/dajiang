# 综合实战（智能笔记助手）

> **Day 6 综合实战**：把 Week 3 前 5 天的全部能力整合为一个「智能笔记发布助手」——输入原始内容，AI 自动完成标题生成 → 摘要提取 → 英文翻译三步流水线，每一步追踪 Token 成本并聚合。

## 场景设计

```
POST /api/chat/smart-note
{ "content": "Spring AI 是 Spring 生态的 AI 框架..." }

          ↓ ChatService.smartNote()

Step 1 ─ .system("你是标题专家")
         └→ "Spring AI：把大模型变成 Spring Bean"
         └→ Token: input=120, output=30

Step 2 ─ .system("你是摘要专家") + PromptTemplate
         └→ "Spring AI 将 LLM 封装为 Spring Bean..."
         └→ Token: input=180, output=60

Step 3 ─ .system("你是翻译助手，中译英")
         └→ "Spring AI is the AI framework in the Spring ecosystem..."
         └→ Token: input=150, output=90

聚合 → SmartNoteResponse {
    title, summary, translation,
    tokens: { inputTokens: 450, outputTokens: 180, totalTokens: 630 }
}
```

## 核心认知：多步 AI 调用链

| 维度 | 单步调用（Day 1-5） | 多步调用链（Day 6） |
|---|---|---|
| 能力 | 一次 LLM 调用做一件事 | 多次 LLM 调用协同完成任务 |
| Token | 单次用量 | 累加聚合 = 链路总成本 |
| 角色 | 一个 System 角色 | 每一步独立 System 角色（精细化约束） |
| 耗时 | 1 次网络往返 | N 次串行 = sum(每次耗时) |
| 可靠性 | 超时+重试保护单次调用 | 超时+重试保护每一步 |
| 测试 | Mock 单次调用 | Mock 聚合方法，验证多字段响应 |

## 代码结构

### 新增 DTO

```
SmartNoteRequest   — 输入：content（原始笔记内容）
SmartNoteResponse  — 输出：title + summary + translation + TokenUsage（聚合）
```

### ChatService.smartNote() 核心逻辑

```java
public SmartNoteResponse smartNote(String content) {
    int totalInput = 0, totalOutput = 0, totalTokens = 0;

    // Step 1: 生成标题
    var titleResp = chatClient.prompt()
        .system("你是标题专家...")
        .user("请为以下内容生成标题：\n" + content)
        .call().chatClientResponse();  // 用 chatClientResponse 拿 Usage
    totalInput += titleResp...getUsage().getPromptTokens();
    // ... 累加

    // Step 2: 生成摘要（PromptTemplate 模板化）
    PromptTemplate t = new PromptTemplate("不超过{maxWords}字：{content}");
    var summaryResp = chatClient.prompt()
        .system("你是摘要专家...")
        .user(t.render())
        .call().chatClientResponse();
    // ... 累加

    // Step 3: 英文翻译
    var translateResp = chatClient.prompt()
        .system("你是翻译助手，中译英，只返回译文...")
        .user(content)
        .call().chatClientResponse();
    // ... 累加

    return new SmartNoteResponse(title, summary, translation,
        new TokenUsage(totalInput, totalOutput, totalTokens));
}
```

### 新端点

```
POST /api/chat/smart-note
  Request:  {"content": "..."}
  Response: {"code": 200, "data": {title, summary, translation, tokens}}
```

## 设计权衡：串行 vs 并行

| | 串行（Day 6 实现） | 并行（CompletableFuture） |
|---|---|---|
| 耗时 | sum(step1+step2+step3) | max(step1, step2, step3) |
| 代码复杂度 | 低（线性代码） | 中（线程池 + join + 异常处理） |
| Token 聚合 | 直观累加 | 需等全部完成再聚合 |
| 调试 | 容易（顺序执行） | 较难（并发交错） |
| 适用场景 | 步骤少（≤3）、调试期 | 步骤多、延迟敏感 |

> Day 6 先用串行建立直觉，并行优化作为扩展练习。
