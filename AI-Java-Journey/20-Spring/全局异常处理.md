# 全局异常处理（@RestControllerAdvice）

> **全局异常处理**用 `@RestControllerAdvice` + `@ExceptionHandler` 拦截所有 Controller 抛出的异常，统一返回格式，避免到处写 try-catch。

## 为什么需要全局异常处理

没有全局异常处理时：
```java
// 每个 Controller 方法都要 try-catch
@PostMapping
public ApiResponse<NoteResponse> create(@Valid @RequestBody CreateNoteRequest request) {
    try {
        Note note = noteService.create(request.title(), request.content());
        return ApiResponse.success(NoteResponse.from(note));
    } catch (MethodArgumentNotValidException e) {
        return ApiResponse.error(400, e.getMessage());
    } catch (RuntimeException e) {
        return ApiResponse.error(404, e.getMessage());
    }
}
```

有了全局异常处理：
```java
// Controller 只管正常逻辑
@PostMapping
public ApiResponse<NoteResponse> create(@Valid @RequestBody CreateNoteRequest request) {
    Note note = noteService.create(request.title(), request.content());
    return ApiResponse.success(NoteResponse.from(note));
}

// 异常统一在 GlobalExceptionHandler 处理
```

## 定义 GlobalExceptionHandler

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 处理 Bean Validation 校验失败
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ApiResponse<Void> handleValidation(MethodArgumentNotValidException e) {
        String msg = e.getBindingResult().getFieldErrors().stream()
                .map(fe -> fe.getField() + ": " + fe.getDefaultMessage())
                .collect(Collectors.joining("; "));
        return ApiResponse.error(400, msg);
    }

    // 处理业务异常（如 Note not found）
    @ExceptionHandler(RuntimeException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ApiResponse<Void> handleRuntime(RuntimeException e) {
        return ApiResponse.error(404, e.getMessage());
    }
}
```

## 注解说明

| 注解                                        | 作用                                                 |
| ----------------------------------------- | -------------------------------------------------- |
| `@RestControllerAdvice`                   | 标记全局异常处理类（= `@ControllerAdvice` + `@ResponseBody`） |
| `@ExceptionHandler(XxxException.class)`   | 拦截指定类型的异常                                          |
| `@ResponseStatus(HttpStatus.BAD_REQUEST)` | 设置 HTTP 状态码                                        |

## 异常处理链路

```
Controller 方法抛异常
    ↓
GlobalExceptionHandler 按异常类型匹配
    ↓
@ExceptionHandler(MethodArgumentNotValidException.class)  ← Bean Validation 失败
@ExceptionHandler(RuntimeException.class)                  ← 业务异常
@ExceptionHandler(Exception.class)                         ← 兜底（可选）
    ↓
返回 ApiResponse.error(code, message)
```

## 自定义业务异常（进阶）

```java
// 定义业务异常类
public class BizException extends RuntimeException {
    private final int code;

    public BizException(int code, String message) {
        super(message);
        this.code = code;
    }

    public int getCode() { return code; }
}

// 在 GlobalExceptionHandler 中处理
@ExceptionHandler(BizException.class)
@ResponseStatus(HttpStatus.BAD_REQUEST)
public ApiResponse<Void> handleBiz(BizException e) {
    return ApiResponse.error(e.getCode(), e.getMessage());
}

// 在 Service 中抛出
public Note getById(Long id) {
    return noteRepository.findById(id)
            .orElseThrow(() -> new BizException(404, "Note not found: " + id));
}
```

## 面试怎么问

- "全局异常处理怎么做？" → `@RestControllerAdvice` + `@ExceptionHandler`
- "`@RestControllerAdvice` 和 `@ControllerAdvice` 区别？" → 前者自带 `@ResponseBody`，返回 JSON
- "校验失败的异常怎么处理？" → 拦截 `MethodArgumentNotValidException`，拼接字段错误信息返回 400
