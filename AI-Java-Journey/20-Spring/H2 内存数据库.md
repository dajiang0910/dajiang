# H2 内存数据库

> H2 是一个轻量级 Java 数据库，支持内存模式和文件模式。开发阶段用内存模式，重启清空，免装 MySQL。

## 为什么开发用 H2

| 特性 | H2 内存模式 | MySQL |
|---|---|---|
| 安装 | 0（Maven 依赖即用） | 需要单独安装 |
| 数据 | 重启清空（方便测试） | 持久化 |
| 速度 | 极快（内存读写） | 需要网络 IO |
| 生产可用 | ❌ | ✅ |

开发阶段用 H2 快速迭代，生产环境换 MySQL 只需改配置——因为 [[面向接口编程]]，Service 一行不用改。

## Spring Boot 4.x 依赖

```xml
<!-- H2 JDBC 驱动 -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

<!-- H2 网页控制台（Spring Boot 4.x 拆成了独立模块） -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-h2console</artifactId>
</dependency>
```

**踩坑**：Spring Boot 4.x 把 H2 控制台从 `spring-boot-autoconfigure` 拆到了 `spring-boot-h2console` 独立模块。只加 `h2` 驱动不够，还需要加控制台模块，否则 `/h2-console` 返回 404。

## application.properties 配置

```properties
# H2 内存数据库连接
spring.datasource.url=jdbc:h2:mem:notesdb
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# 开启 H2 网页控制台
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console
```

## H2 控制台使用

启动应用后浏览器打开 `http://localhost:8080/h2-console`：

| 字段 | 值 |
|---|---|
| JDBC URL | `jdbc:h2:mem:notesdb`（必须和配置一致） |
| User Name | `sa` |
| Password | 留空 |

连接后可以看到 `NOTES` 表，直接执行 SQL。

## 内存模式 vs 文件模式

```
jdbc:h2:mem:notesdb    ← 内存模式，重启数据丢失
jdbc:h2:file:./data/notesdb  ← 文件模式，数据持久化到磁盘
```

| 模式 | URL 格式 | 适用场景 |
|---|---|---|
| 内存 | `jdbc:h2:mem:库名` | 开发、测试、CI |
| 文件 | `jdbc:h2:file:路径` | 本地演示、轻量持久化 |

## 面试怎么问

- "开发环境为什么用 H2？" → 免安装、内存速度快、重启清空方便测试
- "H2 和 MySQL 怎么切换？" → 改 datasource.url 配置，Service 不用改（面向接口编程）
- "Spring Boot 4.x 的 H2 控制台 404？" → 需要加 `spring-boot-h2console` 依赖
