# MOC 与双链（Obsidian 知识管理）

> **MOC**（Map of Content）= 内容地图，用 `[[双链]]` 串联同一主题所有笔记的枢纽笔记。双链是 Obsidian 的核心功能，让笔记从"孤立的岛屿"变成"连通的网络"。

## 它解决什么问题

没有双链时，笔记是线性列表：
```
笔记A
笔记B
笔记C
笔记D
（孤立的岛屿，找关联要靠记忆）
```

有了双链，笔记形成网络：
```
笔记A ←→ 笔记B ←→ 笔记C
      ↕
笔记D ←┘
（连通的网络，关联自动浮现）
```

## 双链语法

```markdown
这是 [[Spring Boot]] 的笔记，和 [[依赖注入(DI)]] 密切相关。
```

- **正向链接**：从当前笔记指向相关笔记（在当前笔记里写 `[[笔记名]]`）
- **反向链接**：被链接的笔记自动知道"谁引用了我"（Obsidian 自动计算）

## MOC 是什么

MOC = Map of Content，**内容地图**。它不是目录，而是用双链串联同一主题所有笔记的枢纽。

### MOC vs 目录

| | 目录（文件夹） | MOC |
|---|---|---|
| 结构 | 树形（一个文件夹一个位置） | 网状（一篇笔记可出现在多个 MOC） |
| 维护 | 手动移动文件 | 只需在 MOC 里加 `[[链接]]` |
| 发现关联 | 难（要跨文件夹找） | 易（反向链接自动显示） |

```markdown
# Week 2 MOC：Spring Boot Web 开发

## 学习轨迹
| 天 | 主题 | 核心笔记 |

## 测试体系

```

## 如何构建知识网络

### 步骤 1：创建笔记时就加双链

```markdown
# 依赖注入(DI)

## 关联
- 上位思想：[[Spring IoC 容器]]
- 深入对比：[[构造器注入 vs 字段注入]]
- 测试应用：[[Mockito 单元测试]] 的 @InjectMocks
```

### 步骤 2：定期创建 MOC 枢纽笔记

```markdown
# Week 2 MOC

## 学习轨迹
Day 1: [[Spring Boot 3 起手]] · [[依赖注入(DI)]]
Day 2: [[三层架构]] · [[面向接口编程]]
...

## 关键概念网络
| 概念 | 连接到 |
|---|---|
| [[依赖注入(DI)]] | → [[构造器注入 vs 字段注入]] → [[面向接口编程]] |
```

### 步骤 3：利用反向链接发现隐藏关联

在 Obsidian 右侧面板打开"反向链接"，可以看到哪些笔记链接到了当前笔记。这能帮你发现隐藏的知识关联。

## 双链的价值

| 场景 | 没有双链 | 有双链 |
|---|---|---|
| 复习时 | 逐篇翻笔记 | 从 MOC 一键跳转 |
| 写新笔记 | 不知道该链到哪 | 反向链接提示"可能相关" |
| 面试准备 | 散落在各处 | MOC 汇总所有要点 |
| 知识迁移 | 孤立记忆 | 网络化理解 |

## 面试怎么问

- "你怎么管理学习笔记？" → MOC + 双链构建知识网络，不是线性笔记
- "Obsidian 和其他笔记工具的区别？" → 双链让知识形成网络，而非树形目录
- "你怎么快速复习？" → 从 MOC 枢纽笔记出发，按主题跳转
