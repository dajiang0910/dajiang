# 文档智能分析综合实战

> Week 4 收网端点：`POST /api/documents/analyze`。上传文档 → Tika 解析 → 文本统计 + AI 元数据提取 + AI 关键句提炼 → 返回综合分析 JSON。首次引入 Spring AI Advisor 做 LLM 调用可观测。

## 它解决什么问题

Day 3-4 的端点要么只解析文本（`/parse`），要么只提取元数据（`/parse-and-extract`）。真实业务场景需要更全面的文档智能分析——不只是"这篇文档讲了什么"，还包括"多长？几个段落？核心句子是哪些？"

`/analyze` 把 Week 4 全部能力整合到一个端点，是知识库"接入层"的完整产品形态。

### 2. 文本统计 vs AI 提取（成本分界线）

| 能力 | 实现方式 | Token 消耗 | 准确度 |
|------|---------|-----------|--------|
| 字符数/句子数/段落数 | 正则 + String 操作 | **0** | 100%（程序计算） |
| 阅读时间 | 字数 ÷ 400 | **0** | 估算 |
| 标题/关键词/分类/摘要 | AI extractV2() | 几百 Token | 受 LLM 质量影响 |
| 关键句提炼 | AI extractKeySentences() | 几百 Token | 受 LLM 质量影响 |

**核心原则：能程序算的绝不调 AI。** 这是成本管控的第一课——每个 `/analyze` 请求有两段 AI 调用（extractV2 + extractKeySentences），比 `/parse-and-extract` 多一次 LLM 调用，Token 成本更高。但关键句提炼是语义理解，正则做不了，该花的 Token 要花。

### 3. SimpleLoggerAdvisor 挂载（Day 5 核心新增）

```java
// 只在 extractKeySentences 方法中挂载
// 不影响全局 ChatClient，不影响 extractV2
chatClient.prompt()
    .system("提取关键句...")
    .user(text)
    .advisors(a -> a.advisors(new SimpleLoggerAdvisor()))
    .call()
    .content();
```

对比 Day 3-4：之前所有 LLM 调用都"裸调"，不记录 prompt/response。挂上 SimpleLoggerAdvisor 后，结合 logback DEBUG 配置，可以看到完整调用日志。

## Java 里怎么落地

```java
// DocumentAnalysisService.analyze()
public DocumentAnalysisResponse analyze(String text, String filename, String mime) {
    // Step 1: 统计（程序计算）
    TextStatistics stats = computeStatistics(text);

    // Step 2: 元数据（AI 提取，复用 Day 2 的 extractV2）
    NoteMetadata metadata = extractService.extractV2(text);

    // Step 3: 关键句（AI 提炼，带 Advisor 观测）
    List<String> keySentences = extractKeySentences(text);

    // Step 4: 组装
    return new DocumentAnalysisResponse(
        filename, mime, metadata, stats, keySentences,
        String.format("约 %d 分钟", stats.estimatedMinutes()));
}
```

## 面试怎么问

- **"你们的知识库接入层分析文档时，哪些用程序算、哪些用 AI？为什么？"** → 文本统计（字符数/句子数/段落数/阅读时间）用程序算，零 Token 成本、100% 准确。标题/关键词/分类/摘要/关键句用 AI，因为涉及语义理解，正则做不了。遵循"能程序算的不调 AI"的成本原则。

- **"这个端点调了几次 LLM？成本怎么估算？"** → 两次：extractV2（元数据提取）+ extractKeySentences（关键句提炼）。qwen-turbo 约 0.008 元/千 token，假设每次 500 token，单次请求约 0.008 元。加上文本统计的 0 成本，整体合理。

- **"如果关键句提取返回空怎么办？"** → 代码里做了 fallback：文本长度 < 200 字时按句号直接切分（零 AI 成本）。AI 返回为空时显示兜底提示"（AI 未能提取关键句）"。
