# Java vs 旧语言

> Java 是"先编译成字节码、再由 JVM 运行"的**静态强类型**语言;核心理念是**一次编译,到处运行**(Write Once, Run Anywhere)。

## 它解决什么问题
理解 Java 和动态脚本语言(Python/JS 等)的根本差异,是从"其他语言转 Java"最该先建立的心智模型。

## 核心要点:运行模型
```
源码 .java --javac编译--> 字节码 .class --JVM--> 各操作系统本地执行
```
- **字节码**:平台无关的中间格式,只认 JVM,不认操作系统。
- 换平台只需换对应平台的 JVM,**代码不用重编** → 跨平台。

## 为什么"先编译再运行"(4 大好处)
- **跨平台**:字节码 + JVM,一次编译到处运行(最核心)。
- **性能**:JVM 有 **JIT(即时编译)**,把热点字节码进一步编成本地机器码,比纯解释执行快。
- **安全**:字节码加载时校验,JVM 是受控运行环境。
- **工程化**:编译期做**类型检查**,很多错误在编译时就暴露(动态语言要运行到那行才报错)。

## 静态 vs 动态类型
| | Java(静态) | Python/JS(动态) |
|--|--|--|
| 类型确定 | 编译期 | 运行期 |
| 变量改类型 | ❌ 不行 | ✅ 可以 |
| 错误暴露 | 编译时 | 运行时 |

## 面试怎么问
- "Java 为什么能跨平台?" → 字节码 + 各平台 JVM。
- "Java 比脚本语言快在哪?" → JIT 即时编译。
