# var 与静态类型

> `var` 让编译器自动推断局部变量的类型;**推断后类型固定,但值仍可改**——这是它和动态类型语言变量的本质区别。

## 它解决什么问题
少写一遍冗长的类型声明(如 `Map<String, List<Person>>`),同时保留 Java 静态类型的安全性。

## 核心要点
- **能用**:方法内局部变量(且声明时必须初始化)、for 循环变量、try-with-resources。
- **不能用**:类成员变量、方法参数、返回值类型、没有初始化的变量。
- **关键纠错**:`var` 固定的是**类型**,不是值。
  ```java
  var count = 1;   // 推断为 int
  count = 5;        // ✅ 值能改
  count = "hi";     // ❌ 类型不能变,编译报错
  ```
- **vs 动态语言**:Python/JS 里变量能随时变成任意类型;Java 的 `var` 只是"省略写类型",底层依然是强类型、编译期检查。

## Java 里怎么落地
Day1 代码:`var people = List.of(...)` —— 编译器推断为 `List<Person>`,后续不能再赋别的类型。

## 面试怎么问
- "`var` 是不是让 Java 变成动态类型了?" → 不是,类型在编译期就定死,只是省略书写。
- "`var` 能用在成员变量上吗?" → 不能。
