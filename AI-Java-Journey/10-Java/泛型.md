# 泛型

> 泛型给容器贴一张"我只装某种类型"的标签,本质是 Java 静态类型在集合上的延续。

## 它解决什么问题
动态语言的 list 啥都能塞(`[1, "hi", true]`),错误要运行时才暴露。泛型让编译器在**编译期**就帮你挡住装错类型,并免去取值时的强制转换。

## 核心要点
- 写法:`List<String>`、`Map<String, Integer>` —— 尖括号里是"装什么"。
- **好处一:编译期类型检查**
  ```java
  List<String> names = new ArrayList<>();
  names.add("Ada");
  names.add(123);   // ❌ 编译报错,运行前就拦下
  ```
- **好处二:取出免强转**
  ```java
  String first = names.get(0);   // 直接是 String,无需 (String) 强转
  ```
- 裸 `List`(不写泛型)什么都能塞,取出要手动强转,易出错 → 别用。

## Java 里怎么落地
Day2:`List<Doc>`、`HashMap<String,Integer>`、`HashSet<String>` 全都带泛型。

## 面试怎么问
- "泛型有什么好处?" → 编译期类型检查 + 免强制转换。
